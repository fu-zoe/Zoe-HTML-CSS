Chapter 3 Text

Text, Semantics & Entities

Block-level vs inline elements, character entities, and everyday text semantics — with live previews.

Block-level vs inline

Block-level (structure)

  • Starts on a new line by default.
  • May contain block-level and inline elements.
  • Think of them as containers/structure.

Inline (text modifiers)

  • Flows within a line; does not break the line by itself.
  • May contain text or other inline elements.
  • Think of them as inline content or text styling.

Common elements by category

Quick reference: block-level vs inline elements
Block-level Elements Inline Elements
<p> (paragraph) <em> (emphasis, italic)
<h1>–<h6> (headings) <strong> (strong importance, bold)
<div> (generic container) <a> (anchor, hyperlink)
<ul>, <ol> (lists) <abbr> (abbreviation)
<li> (list item) <span> (generic inline container)
<table>, <tr>, <td> <img> ✓ (image)
<form> (form container) <code> (inline code)
<section>, <article> <br> ✓ (line break)
<blockquote> <q> (inline quotation)
<hr> ✓ (thematic break) <input> ✓ (form input)

✓ = void element (no closing tag).

HTML character entities

  • &nbsp; = non-breaking space (no automatic wrap)
  • &lt; = < (less-than)
  • &gt; = > (greater-than)
  • &amp; = & (ampersand)
  • &copy; = © (copyright)
