Document anatomy

A valid HTML page has <!doctype html> followed by <html> → <head> → <body>.

  • Exactly one page-level <h1>.
  • <html lang="en"> sets the language.
  • Keep structure semantic and predictable.
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello</h1>
  </body>
</html>

Minimal modern template

Use a responsive viewport and a short, unique title.

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Page Title</title>
  </head>
  <body>
    <h1>Heading</h1>
  </body>
</html>

Tags vs. Elements

An element = opening tag + content + closing tag. Some are void elements (no closing tag/content) but still count as elements. 

<p>Hello</p>        <!-- element with content -->
<img src="cat.jpg" alt="A cat">  <!-- void element -->

Common void elements: img, br, hr, meta, link, input, source, track, area, col, embed, param, wbr.

Attributes 101

  • Syntax: name="value" (always use quotes).
  • Global: id, class, title, lang, dir, style, hidden, tabindex, data-*, aria-*.
  • Boolean: required, disabled, checked can be written by name only. 
<input id="email" class="u-w-full" type="email" required aria-label="Email">

Comments & Entities

<!-- This is a comment. It won't render. -->
Use entities for special characters:
<  &lt;     >  &gt;     &  &amp;     "  &quot;     '  &apos;

Language & semantics

  • Language on <html> improves accessibility and search.
  • Prefer semantic tags over generic <div> where possible. 
<main>
  <article>
    <h1>A blog post</h1>
    <p>...content...</p>
  </article>
</main>

Basic nesting rules

  • <li> must be a direct child of <ul> or <ol>.
  • Do not nest block containers inside <p>.
  • Close tags in the reverse order they were opened.
<ul>
  <li>Good</li>
</ul>

<p>Bad: <h2>Heading</h2></p> <!-- invalid nesting -->

Validate early

Validate your HTML during development to catch missing attributes, invalid nesting, and other syntax issues. Use the W3C Markup Validation Service.

Local preview: localhost & port

127.0.0.1 (aka localhost) points to your own machine. The number after the colon is the port your dev server uses. 

http://127.0.0.1:5500/index.html
# 127.0.0.1 → your computer
# 5500      → the server's port (may vary)

Quick checklist

  • <!doctype html> present
  • <html lang> present
  • One <h1>; headings descend in order
  • Meaningful <title> & description
  • Semantic layout landmarks
  • No invalid nesting; required attributes present
