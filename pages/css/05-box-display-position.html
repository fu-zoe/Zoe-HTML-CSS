Box Model, Display & Position

Margin / border / padding / content & box-sizing; block vs inline vs inline-block; relative / absolute / fixed / sticky; z-index.

Box model anatomy

Margin (orange) → Border (gray) → Padding (green) → Content (blue). For visualization below, margin is simulated with a wrapper (real margins can’t be colored).

Shorthand cheatsheet

/* margin (clockwise) */
margin: top right bottom left;
margin: top right-left bottom;
margin: top-bottom right-left;
margin: 3rem auto;         /* centers block horizontally */

/* border: width || style || color */
border: 2px dashed red;
border-radius: 50%;         /* circle if box is square */

/* padding mirrors margin shorthands */
padding: 1rem 2rem;

Outline vs border

Outline doesn't affect layout (good for focus). Use outline-offset to pull it away from the border. 

.demo-btn:focus-visible {
  outline: 3px solid var(--color-accent);
  outline-offset: 4px;
}

Display

  • Block takes full width (e.g., p, ul).
  • Inline flows in text; cannot set height or vertical margins.
  • Inline-block behaves like inline but accepts width/height and vertical margins.

Notes

  • relative: moves from its original spot; original space is kept.
  • absolute: positioned to the nearest positioned ancestor; falls back to initial containing block if none.
  • fixed: fixed to viewport; remains on scroll (not demoed here to avoid conflicting with page navbar).
  • sticky: toggles between relative and fixed within its scroll container.
  • z-index: higher numbers stack above lower ones. 
.pos-stage { position: relative; max-height: 220px; overflow: auto; }
.pos--rel   { position: relative; left: 12px; top: -6px; }
.pos--abs   { position: absolute; right: 8px; bottom: 8px; }
.pos--sticky{ position: sticky; top: 4px; }
