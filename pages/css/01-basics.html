CSS Chapter 1

Basics & Cascade (Start Here)

Three ways to add CSS (external / internal / inline), when to use each, and how the cascade decides “who wins”.

1) Stylesheet placement

External stylesheet

Best for real projects: clean separation, caching & reuse. 

<link rel="stylesheet" href="css/style.css">

This text is styled by external CSS (this page’s .css file).

Internal stylesheet

Use sparingly for page-local tweaks or demos.

<style>
  .demo-internal .demo-text { color: blue; }
</style>

This text is styled by an internal <style> block.

Inline styles

Avoid in production; okay for tiny tests or quick overrides. 

<p style="color: red">This is red text</p>

This text is styled by an inline style.

2) Cascade & specificity (who wins?)

Cascade order

  1. Origin & importance: user agent < author < !important
  2. Specificity: inline > ID > class/attr/pseudo-class > element/pseudo-element
  3. Source order: later rules override earlier ones (if above tie)

Specificity scores

/* think: (inline, id, class, element) */
p                 /* 0,0,0,1 */
p.note            /* 0,0,1,1 */
#intro            /* 0,1,0,0 */
[aria-current]    /* 0,0,1,0 */
a:hover           /* 0,0,1,1 */
h1::first-line    /* 0,0,0,2 */
style="..."       /* 1,0,0,0 */

Tiebreak demo

Class vs element: class wins; later rule wins on equal specificity. 

/* earlier */
p { color: slateblue; }
/* later with same specificity (element) overrides */
p { color: seagreen; }
/* class beats element */
.note { color: tomato; }

I end up seagreen…

…but with .note class I turn tomato.
