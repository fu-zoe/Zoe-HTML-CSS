Selectors & Pseudo

Element, class, attribute, id, universal — then cascade & specificity. Plus interactive pseudo-classes and pseudo-elements for real-world UI.

1) Core selectors

Element / group / descendant

Rule of thumb: comma = “either”; space = “inside (descendant)”.
Note: h1 h2 means “an h2 inside an h1” — that’s practically never valid; use article h2 or p span as realistic examples. 

/* element: match all <p> */
p { margin-block: 1rem; }

/* group (comma): style both h1 and h2 */
h2, h3 { color: var(--color-primary); }

/* descendant (space): only <span> inside <p> */
p span {
  color: var(--color-accent);
  font-weight: 600;
}

/* FYI: h1 h2 = an <h2> inside an <h1> (rare/invalid in practice).
   Prefer: article h2  or  p span. */

Section title: <h2>

Subheading: <h3>

A paragraph with a descendant span used for highlighting. Prefer a semantic class (e.g., .highlight) when your intent is “highlight”, not “any span in p”.

Class selector

<p class="note">Marked text</p>
/* CSS */ .note { color: slategray; font-style: italic; }

This paragraph is styled by a .note class.

Attribute selectors

Match elements by attributes — works for native attributes (e.g., type) and custom data-* attributes. Tip: custom attributes must start with data- to validate and be usable in CSS/JS. 

<input type="text" placeholder="text field">
<button data-role="admin">Admin</button>
<button data-role="user">User</button>

/* CSS */ 
/* Native attribute (exact match) */
input[type="text"] { outline: 2px dashed var(--color-primary); }

/* Custom data- attribute (exact match) */
button[data-role="admin"] {
  background: color-mix(in srgb, var(--color-primary) 15%, transparent);
}

/* Presence: any element that has data-role */
[data-role] { border: 3px solid var(--color-border); }

/* Starts-with / ends-with / contains (use when helpful) */
[data-role^="ad"] { font-weight: 700; }   /* starts with "ad" */
[data-role$="er"] { text-decoration: underline; } /* ends with "er" */
[data-role*="se"] { letter-spacing: .2px; } /* contains "se" */

Use attribute selectors for structural hooks. Prefer classes for pure styling intent (.btn--primary, .is-active, etc.).

ID selector (rare)

<p id="second">Hi</p>
/* CSS */ #second { border: 1px solid var(--color-border); padding: .25rem .5rem; }

First (does not has id)

Second (matched by #second)

IDs should be unique; prefer classes for reusable styling.

Universal selector

.universal-demo > * { outline: 1px dotted var(--color-border); padding: .25rem; }
Every child gets outlined via *

2) Cascade, specificity, inheritance

Specificity & source order

/* same specificity (element) — later wins */
p { color: seagreen; }
p { color: steelblue; }

/* class beats element */
.note { color: tomato; }

Equal specificity: later rule wins (→ steelblue)

Class beats element (→ tomato)

Inheritance & form controls

/* Text properties inherit to children, but many form controls don't.
   In this project, base.css sets:
   button, input, textarea, select { font: inherit; } */

Parent defines a custom font-size; the button inherits.

Avoid !important

.chip { background: var(--color-surface); }
.chip.urgent { background: tomato !important; }  /* last resort */
Normal Urgent (!important)

Prefer good structure & specificity strategy over !important.

3) Pseudo-classes

Interactive states

.chip:is(:hover, :focus) { transform: translateY(-1px); }
.chip:active { transform: translateY(0); }
Hover / Focus / Active

:target

<a href="#target-box">Focus the box</a>
#target-box:target { outline: 2px solid var(--color-accent); }
Jump & highlight
Target me via URL hash

:is() & :where() 

/* :is merges multiple selectors (keeps highest specificity among them) */
.menu :is(a, button):is(:hover, :focus) { text-decoration: underline; }

/* :where has zero specificity — great for default spacing */
.cards :where(h3, p) { margin: 0; }
Link

Zero-specificity spacing

Using :where() to avoid fights.

Link states

a:any-link { color: var(--color-primary); } /* both :link and :visited */
a:focus { outline: 2px solid var(--color-accent); outline-offset: 2px; }

External link · Internal link

Browsers restrict styling of :visited for privacy; expect limited effects.

Attribute presence & :not()

img[alt] { outline: 2px solid #69b; }
img:not([alt]) { outline: 2px solid tomato; }

:nth-child vs :nth-of-type

:nth-child(n) counts an element among all siblings. :nth-of-type(n) counts only among siblings of the same tag. (Both are 1-based.) 

.mix :nth-child(2) { background: color-mix(in srgb, var(--color-primary) 15%, transparent);}
.mix p:nth-of-type(2) {background: color-mix(in srgb, var(--color-accent) 20%, transparent);}

/* Tips */
li:nth-child(even) { /* every even item */ }
p:nth-of-type(3n) {  /* every 3rd <p> */ }

p[1]

span[2] ← selected by :nth-child(2)

p[2] ← selected by p:nth-of-type(2)

4) Pseudo-elements

::before & ::after 

.tag::before { content: "★ "; }
.tag::after  { content: " · badge"; }

Generated content appears before/after.

::first-letter & ::first-line 

.dropcap::first-letter { font-size: 180%; font-weight: 700; }
.excerpt::first-line { text-transform: uppercase; letter-spacing: .03em; }

Drop caps are classic for intros.
