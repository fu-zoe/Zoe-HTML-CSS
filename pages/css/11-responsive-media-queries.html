Responsive & Media Queries

Use the viewport meta tag, write media queries (min/max/range, orientation, aspect-ratio, high DPI) and respect user preferences (dark mode, reduced motion).

1) Viewport meta (enable responsive layout)

Add this inside <head> so CSS pixel width maps to the device width on phones/tablets: 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

2) Media query syntax

/* Pattern */
@media <media-type> and (<feature>) { /* rules */ }

/* Common types (type is optional): screen | print */
@media screen and (min-width: 48em) { /* ... */ }

/* Features */
@media (min-width: 481px) { /* width up from 481 */ }
@media (max-width: 480px) { /* width up to 480 */ }
@media (orientation: landscape) { /* wide orientation */ }
@media (min-aspect-ratio: 16/9) { /* at least 16:9 */ }
@media (min-resolution: 2dppx) { /* high DPI screens */ }

/* User preferences */
@media (prefers-color-scheme: dark) { /* dark mode styles */ }
@media (prefers-reduced-motion: reduce) { /* limit motion */ }

/* Modern range syntax (fallback below if needed) */
@media (768px <= width <= 1024px) { /* tablet-ish range */ }
/* Fallback: */
@media (min-width: 768px) and (max-width: 1024px) { /* ... */ }

3) Breakpoints (resize the window)

≤ 480px ≥ 481px ≥ 768px (tablet+) 768–1024px (range) 
/* Example breakpoints using em for zoom-resilience */
@media (max-width: 30em) { /* ≤ 480px */ }
@media (min-width: 30.063em) { /* ≥ 481px (approx) */ }
@media (min-width: 48em) { /* ≥ 768px */ }
@media (48em <= width <= 64em) { /* 768–1024px (modern) */ }
@media (min-width: 48em) and (max-width: 64em) { /* fallback */ }

4) Orientation & aspect-ratio

orientation: portrait orientation: landscape min-aspect-ratio: 16/9 max-aspect-ratio: 4/3 
@media (orientation: landscape) { /* ... */ }
@media (orientation: portrait)  { /* ... */ }
@media (min-aspect-ratio: 16/9) { /* screens ≥ 16:9 */ }
@media (max-aspect-ratio: 4/3)  { /* screens ≤ 4:3  */ }

5) High DPI (retina-like)

min-resolution: 2dppx (retina) standard DPI

Note: On non-retina displays the “retina” flag will stay hidden (that’s expected).

6) User preferences

prefers-color-scheme

This card follows your system color scheme (light/dark).

System color preview
@media (prefers-color-scheme: dark) {
  .pref-card { background: #111; color: #eee; }
}

prefers-reduced-motion

The dot animates unless the user prefers reduced motion. 

@media (prefers-reduced-motion: reduce) {
  .ball { animation: none; }
}

7) Combine conditions

AND: (min-width: 768px) & (orientation: landscape) OR: (max-width: 480px), (orientation: portrait) Range: 48em ≤ width ≤ 64em 
@media screen and (min-width: 768px) and (orientation: landscape) { /* ... */ }
@media screen and (max-width: 480px), (orientation: portrait) { /* ... */ }
@media (48em <= width <= 64em) { /* modern range */ }
@media (min-width: 48em) and (max-width: 64em) { /* fallback */ }

8) Patterns

  • Prefer em breakpoints (@media (min-width: 48em)) for better zoom resilience.
  • Start mobile-first (grow styles with min-width queries).

Quick snippets

/* Hide on small screens */
@media (max-width: 30em) {
  .sidebar { display: none; }
}

/* Scale text after tablet */
@media (min-width: 48em) {
  body { font-size: 1.125rem; }
}

/* Landscape-like control (short viewports) */
@media (max-height: 26.5em) and (min-aspect-ratio: 7/4) {
  /* ... */
}
