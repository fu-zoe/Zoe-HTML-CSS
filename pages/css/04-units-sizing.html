Units & Sizes

Absolute vs relative units, percentages & CSS numeric types, rem/em/ch, viewport units, and box-sizing resets.

1) Absolute length units

On screens, “physical” units are mapped to CSS pixels by the UA; use them sparingly. Common ones: px (CSS pixel), pt (1/72 in), pc (12pt), in, cm, mm, Q (quarter-mm). 

/* Squares sized in different absolute units (visual reference only) */
.abs-in { width: 1in; height: 1in; }
.abs-cm { width: 2cm; height: 2cm; }
.abs-mm { width: 20mm; height: 20mm; }
.abs-pt { width: 72pt; height: 72pt; }   /* ≈ 1 inch */
.abs-pc { width: 6pc;  height: 6pc;  }   /* 1pc = 12pt; 6pc ≈ 1 inch */
.abs-px { width: 96px; height: 96px; }   /* 96px ≈ 1 inch (typical mapping) */

2) Percentages & CSS numeric types

CSS uses data types like <integer> (e.g. z-index: 10), <number> (unitless, e.g. line-height: 1.6), <dimension> (= <number> + unit, e.g. 2rem), and <percentage> (e.g. width: 80% relative to containing block). 

.bar { width: 50%; }          /* % relative to parent content box */
.opacity { opacity: .5; }     /* <number> (unitless) */
.z { z-index: 10; }           /* <integer> */
.pad { padding: 1.25rem; }    /* <dimension> */
50%

3) Relative units: rem & em

rem is based on the root (<html>) font-size; em is based on the element’s computed font-size. On font-size, em compounds with inheritance; on other properties (padding/margin/width) it uses the element’s own font-size. 

:root { font-size: 16px; }     /* 1rem = 16px */
.scope { font-size: 125%; }    /* children now 20px (1.25 × 16) */
.box-rem { padding: 1rem; }    /* always 16px */
.box-em  { padding: 1em;  }    /* here = 20px, because element font-size is 20px */
padding: 1rem (16px)
padding: 1em (20px here)

4) ch (width of “0”)

Useful for readable text widths (e.g. 60–75ch). If glyph metrics are unknown, UAs assume 0.5em × 1em. 

.prose-30 { max-width: 30ch; }
.prose-60 { max-width: 60ch; }
30ch — shorter lines feel tighter but are easy to scan on small viewports.
60ch — common target for comfortable reading at desktop sizes.

5) Viewport units (vw / vh)

1vw = 1% of viewport width; 100vw includes the scrollbar and can overflow. Prefer width: 100% for layout inside containers. 

.box-100p { width: 100%; }      /* safe: relative to parent */
.box-100vw { width: 100vw; }    /* may overflow parent and cause horizontal scroll */

6) Reset & box-sizing

A minimal reset often sets margins to zero and switches to border-box so widths include padding/border. Our project’s base.css already does this; below shows the difference with content-box. 

/* Common reset */
*, *::before, *::after { box-sizing: border-box; }
html, body { margin: 0; }

/* Compare: same declared width, different real size */
.cb  { box-sizing: content-box; width: 200px; padding: 20px; }
.bb  { box-sizing: border-box;  width: 200px; padding: 20px; }
