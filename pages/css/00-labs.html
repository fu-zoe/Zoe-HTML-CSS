Project Summary (Mistakes & Key Points)

Quick notes I wish I had on day one — nav patterns, section/article IDs, CSS reset & utilities, sticky header/nav, main layout, profile card sizing, and mobile-first media queries.

1) HTML — Navigation

  • You don’t have to use a list in <nav> — a row of <a> works fine for simple sites.

Simple (anchors only)

<nav aria-label="Primary">
  <a href="#intro">Intro</a>
  <a href="#cards">Cards</a>
  <a href="#contact">Contact</a>
</nav>

List-based (also valid)

<nav aria-label="Primary">
  <ul>
    <li><a href="#intro">Intro</a></li>
    <li><a href="#cards">Cards</a></li>
    <li><a href="#contact">Contact</a></li>
  </ul>
</nav>

2) HTML — Sections & IDs

  • Use the same class for repeated patterns (e.g. class="card").
  • Give each section a unique id to enable nav jumps. 
<article id="about" class="card">...</article>
<article id="skills" class="card">...</article>

3) CSS — Reset

  • Start clean and let CSS control sizing.
  • Images: override any HTML width/height via CSS so they scale with the layout. 
*, *::before, *::after { box-sizing: border-box; }
* { margin: 0; padding: 0; }
img { display: block; max-width: 100%; height: auto; }  /* CSS rules win over HTML width/height */

4) CSS — Utility class

  • .nowrap — wrap a short phrase to prevent awkward line breaks. 
<p>Keep <span class="nowrap">product names</span> intact.</p>
/* CSS */ .nowrap { white-space: nowrap; }

5) CSS — General classes

Page basics

html { scroll-behavior: smooth; }

body {
  min-height: 100vh;
  /* fonts, background-color / background-image, layout display, etc. */
}

Header / footer (sticky)

header { position: sticky; top: 0; }
footer { position: sticky; bottom: 0; }
/* color, text-align ... */

Nav

nav {
  display: flex;
  gap: .75rem;
  /* color, font, padding, border, pseudo-classes ... */
}

Main layout

main {
  display: flex;
  flex-direction: column;
  gap: 1rem;       /* or a token variable */
  padding: 1rem;
  align-items: start;
}

6) Profile Card

  • scroll-margin-top — leave space for a sticky header when anchored.
  • Sizing with min()/max()/clamp() or nested calc().

Code

.card {
  scroll-margin-top: calc(var(--nav-h) + 12px);
  width: min(100%, 350px); /* cap width at 350px, never exceed container */
}

/* Advanced: compare two expressions and take the smaller one */
.card {
  width: min(100% - 2rem, calc(50% + 100px));
  /* calc(): do math across units (%, px, rem...) */
}

7) Media Queries

  • Use mobile-first with min-width. Larger breakpoints inherit smaller rules unless you override them. 
/* Base (mobile) */
.card-grid { grid-template-columns: 1fr; }

/* Wider screens enhance layout */
@media (min-width: 600px) {
  .card-grid { grid-template-columns: 1fr 1fr; }
}
@media (min-width: 900px) {
  .card-grid { grid-template-columns: repeat(3, 1fr); }
}
